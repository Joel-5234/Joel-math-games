# Issue #012: Multiple Choice Options Not in Alphabetical Order

**Date:** 2025-11-15T10:58:47  
**Status:** Fixed  
**Priority:** Medium

## Description
Multiple choice options in Challenge mode (and potentially other modes) are displayed in random order instead of alphabetical order (A, B, C, D). Options appear shuffled (e.g., C, B, D, A) which makes it confusing for students.

## Root Cause Analysis
1. **Random Shuffling**: Options are shuffled using `.sort(() => Math.random() - 0.5)` in question generators
2. **No Sorting Before Rendering**: The `renderRadioOptions()` function renders options in whatever order they're provided
3. **Correctness Tracking**: Correctness is properly tracked via `dataset.correct` attribute, but display order is random

## Solution
1. Created `sortOptionsByLabel()` helper function to sort options alphabetically by label (A, B, C, D)
2. Updated `renderRadioOptions()` to sort options alphabetically before rendering
3. This ensures options are always displayed in order A, B, C, D regardless of how they were shuffled in generators

## Changes Made
- **app.js**: 
  - Added `sortOptionsByLabel()` function to sort options alphabetically
  - Modified `renderRadioOptions()` to sort options before rendering
  - This applies to all modes (Practice, Challenge, Session) since all use `renderRadioOptions()`
