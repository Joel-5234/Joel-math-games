# Issue #009: Challenge Mode Not Available for New Problem Types

**Date:** 2025-11-12 21:12:26  
**Status:** Fixed  
**Last Updated:** 2025-11-12 21:12:26

## Description
Challenge mode is not working for the new problem types (Intercepts, Rate of Change, Linear Functions, Standard Form). When users select these tabs and try to start a challenge, the challenge doesn't generate questions correctly or fails to start.

## Root Cause Analysis
The `startChallenge()` function was using the tab's `data-tab` attribute directly as the problem type, but there's a mismatch between tab names and generator keys:

**Tab data-tab values:**
- `intercepts` (tab)
- `rateofchange` (tab)
- `linearfunction` (tab)
- `standardform` (tab)

**Generator keys in generateChallengeQuestions():**
- `intercept` (generator)
- `rateOfChange` (generator)
- `linearFunction` (generator)
- `standardForm` (generator)

When `startChallenge()` passed `linearfunction` directly, it couldn't find the `linearFunction` generator, causing the challenge to fail or generate empty question sets.

## Solution
Created a mapping function `mapTabToProblemType()` to convert tab names to the correct generator keys:
- `intercepts` → `intercept`
- `rateofchange` → `rateOfChange`
- `linearfunction` → `linearFunction`
- `standardform` → `standardForm`

Updated `startChallenge()` to use this mapping function before calling `initializeChallenge()`.

## Changes Made

### app.js
- Added `mapTabToProblemType()` function to map tab names to generator keys
- Updated `startChallenge()` to use the mapping function
- Ensures all 8 problem types work correctly in Challenge mode

## Testing

- All new problem types should now work in Challenge mode

## Notes

- New problem types had mismatched names, causing the issue
- Mapping function provides a single source of truth for tab-to-type conversion
