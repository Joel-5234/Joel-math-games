
## Files and Folders
./
    - config.json or appsettings.json (depending on the tech stack; application configuration file)
    - .cursorrules: rules for the ai coding assistant
./.cursor/commands/
    - fix.md: content of ./prompts/fix.md
    - new.md: content of ./prompts/new.md
    - proceed.md: content of ./prompts/proceed.md
- ./docs
    - prd.md: product requirements document
    - ideas.md: user's idea/input
    - readme.md
- ./docs/tasks
    - tasks-index.json and tasks-<milestone_id>.md. Tasks grouped by milestones
- ./docs/issues
    - issues-index.json: index of the list of issues.  includes short description, status, filename, id, date/timestamp.
    - issue-<id>-<datetime>.md
- ./dist
    - the files to deploy to production environment
- ./logs
    - app.log: Logs generated by the application. 
    - activity.log: log of ai coding assistant activity
- ./scripts
    - start-dev: start the development environment
    - stop-dev: stop the development environment
    - check-status: checks the status of the environment
    - build: builds the environment
- ./data
    - initially empty
    - any data to use as seeding or importing.
- ./ref
    - initially empty
    - References for later use 
- ./tests
    - /ui: UI tests.  Use Playwright for UI tests
    - /api: api tests
- ./prompts
    - fix.md: prompt to create/resolve an issue
    - new-idea.md: prompt to register an idea into tasks/milestones
    - proceed.md: prompt to complete next, open milestone
    - test.md: prompt to test the system
    - continue.md: ai coding assistant may have stalled. Use this to ask reset or start a new chat window.
